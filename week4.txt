Enter Bucket name (Example: my-public-bucket-123)

What is Versioning?

Versioning in Amazon S3 keeps multiple versions of the same object.

Step 1: Enable Versioning

Open S3

Select bucket

Go to Properties

Scroll to Bucket Versioning

Click Edit

Select Enable

Click Save changes

‚úî Versioning enabled

Step 2: Upload File

Upload report.pdf

‚úî Version ID automatically generated

Step 3: Re-upload Same File

Upload report.pdf again

‚úî Old version preserved
‚úî New version becomes current

Step 4: Delete Object

Click Delete

‚úî Delete Marker created
‚úî Old versions still available
‚úî Can restore anytime

üîπ PART 3: Cross-Region Replication (CRR)
What is CRR?

CRR replicates data between different AWS regions.

Prerequisites (VERY IMPORTANT)

‚úî Versioning enabled on both buckets
‚úî Buckets in different regions

Step 1: Create Destination Bucket

Go to S3 ‚Üí Create bucket

Enter name: dest-crr-bucket

Select different region

Create bucket

Enable Versioning

Step 2: Create Replication Rule

Open Source bucket

Go to Management tab

Click Replication rules

Click Create replication rule

Step 3: Configure Rule

Rule name: academy-crr-rule

Status: Enabled

Apply to: All objects

Step 4: Choose Destination

Select Another bucket

Choose destination bucket

Confirm region is different

Select Use AWS-managed role

Click Save

‚úî Replication active

Step 5: Test Replication

Upload image.png to source bucket

Open destination bucket

‚úî Object appears automatically

‚úî Version ID present

‚úî CRR working successfully
